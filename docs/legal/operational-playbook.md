# マーケットプレイス運用プレイブック

## 1. 通常注文

1. `orders/paid` の受信と店舗別台帳計上を確認する。
2. 売主不明、商品紐付け不明又は店舗状態異常の場合は、自動補正せず案件を作る。
3. 店舗は注文管理から配送会社と追跡番号を登録する。
4. Shopifyの注文及び店舗画面の配送状態が一致することを確認する。
5. 保留期間経過後に、返金・紛争・準備金を差し引いた出金可能額を計算する。

## 2. 返金・キャンセル

1. 購入者の申請を受け付け、受付日時を固定する。
2. 注文、購入者、商品、売主、支払状態及び発送状態を照合する。
3. メール不一致又は注文不明は本人確認待ちに隔離する。
4. 店舗の意見と証拠を収集する。
5. 商品代金、初回通常送料、追加配送費、返送費及び価値減少額を分けて判断する。
6. 管理者が返金又はキャンセルを実行する。
7. Shopify webhookによる台帳反映を確認する。
8. 店舗責任がある場合だけ、別途責任案件と精算調整を作る。

## 3. 商品事故・リコール

### 初動

- 対象商品の新規販売を停止する。
- 商品ID、ロット、注文、購入者、売主及び事故内容を保全する。
- 人身事故のおそれがある場合は、使用中止を案内する。
- 売主、製造者、輸入者、保険会社及び必要な行政窓口へ連絡する。

### 調査

- 商品写真、説明、許認可、検査資料及び仕入経路を取得する。
- 同一商品の販売件数と購入者を特定する。
- 運営、売主、製造者、配送会社及び購入者の行為を分けて評価する。
- 責任割合と確定金額を案件へ記録する。

### 終了

- 回収、交換、返金及び購入者通知の完了を確認する。
- 掲載再開には再審査を必要とする。
- 費用負担は証拠に基づき確定し、精算調整又は直接請求を行う。

## 4. 偽造品・知的財産権侵害

1. 申立てを記録し、対象商品を一時停止する。
2. 申立人の権利及び対象商品の対応関係を確認する。
3. 売主へ真正性、仕入先及び販売権限の証拠提出を求める。
4. 証拠不十分又は侵害が明らかな場合は掲載を停止し、再出品を防ぐ。
5. 対象注文、返金、購入者通知及び必要な情報開示を処理する。

## 5. チャージバック・不正決済

1. 決済会社の期限を確認する。
2. 注文、配送、追跡、本人連絡及び商品説明を証拠として提出する。
3. 調査中は対象額を準備金又は出金保留にする。
4. 責任確定前に売上を相殺しない。
5. 結果確定後、購入者不正、店舗不履行又は運営上の問題を区分する。
6. 店舗責任が確定した場合だけ精算調整を適用する。

## 6. 店舗失踪・支払不能・負残高

1. 新規出品、受注及び出金を停止する。
2. 未発送注文、返品中商品、未確定返金及びチャージバックを集計する。
3. 準備金と未精算売上を確保する。
4. 購入者保護を優先して、注文履行又は返金方針を決める。
5. 確定債務を未精算売上と相殺し、不足分を直接請求する。
6. 回収不能見込みは会計・税務上の処理を確認する。

## 7. 個人情報漏えい

1. 影響するアカウント、秘密鍵、APIキー及びセッションを停止・更新する。
2. ログを保全し、閲覧・流出したデータと人数を確定する。
3. 連絡責任者を一本化し、店舗が独自に不正確な案内をしないようにする。
4. 法定報告、本人通知、決済会社、Shopify、Render及び保険会社への連絡要否を判断する。
5. 復旧後、原因、再発防止、未対応事項及び完了日を記録する。

## 8. 外部サービス障害

### Shopify又は決済

- 新規注文の成否が不明な場合、購入者へ再決済を促す前に注文を照会する。
- webhook遅延時は二重台帳を作らず、イベントIDで再処理する。

### Render又はDB

- デプロイ停止、直前コミット、migration状態及びDB接続を確認する。
- DB復元は本番へ直接行わず、別環境で復元確認してから切り替える。

### Resend

- Outboxを保持し、復旧後に冪等キー付きで再送する。
- 同じ通知を手作業とワーカーの両方から送らない。

## 9. エスカレーション基準

| 事象 | 初動期限 | 承認者 |
| --- | --- | --- |
| 人身事故・重大製品事故 | 即時 | 運営責任者 |
| 個人情報漏えいのおそれ | 即時 | 運営責任者・セキュリティ担当 |
| 行政・裁判所・警察からの照会 | 当日 | 運営責任者・専門家 |
| チャージバック | 決済会社期限より前 | 精算担当 |
| 通常返金・撤回 | 公開ポリシーの期限内 | サポート担当 |
| 店舗負残高 | 次回出金前 | 精算担当・運営責任者 |

## 10. 監査証跡

判断を伴う処理には、担当者、日時、証拠、決定理由、対象注文、対象店舗、金額、通貨及び実行結果を残します。メールやチャットだけで責任確定・相殺を行いません。
